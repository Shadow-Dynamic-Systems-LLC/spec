# Regulated Release Checklist

**Role:** Verifier  
**Used in:** `sds-governed` tier  
**Gate:** Pre-release (before deployment to Lighthouse execution envelope)

All items must pass before a governed-tier change is deployed. Items marked CRITICAL block deployment; items marked ADVISORY produce warnings. Run `/sds:check-drift` as the final step — it must pass with no violations.

---

## Traceability Completeness (CRITICAL)

- [ ] `/sds:traceability-check` passes with no errors
- [ ] Every `<requirement>` is linked to at least one `<scenario>` or `<evidence>`
- [ ] Every `<control>` is linked to at least one `<requirement>` and one `<evidence>`
- [ ] Every `<decision>` in `architecture.md` is linked to at least one `<requirement>`
- [ ] `traceability.md` was regenerated after the last modification to any tagged artifact

## Simulation Clearance (CRITICAL)

- [ ] `simulation.md` exists and was generated by `/sds:simulate-capability`
- [ ] `simulation.md` contains no open CRITICAL findings
- [ ] All CRITICAL findings from prior simulation runs are resolved or have accepted mitigations recorded in `risk.md`
- [ ] Capability utilization is documented and dead zones are either resolved or explicitly accepted

## Capability Surface Currency (CRITICAL)

- [ ] `capability_surface.md` was regenerated after the last modification to `capability.yaml`
- [ ] `capability_surface.md` was regenerated after the last modification to `controls.md`
- [ ] No dead zones in `capability_surface.md` are unresolved without explicit acceptance

## Drift Detection (CRITICAL — requires Phase 4 tooling)

> These items require `/sds:check-drift` and `/sds:compile-baml`, implemented in Phase 4.
> Until those tools are available, mark this section BLOCKED and do not sign off on individual items.

- [ ] `/sds:check-drift` passes with no violations
- [ ] `validation_hash` in `capability.yaml` matches current BAML function output
- [ ] All guarantees in `capability.yaml` hold on current execution output
- [ ] BAML function stubs were not manually modified (generated by `/sds:compile-baml`)

## Evidence Completeness (CRITICAL)

- [ ] All pre-approval evidence items in `validation.md` are satisfied
- [ ] All pre-merge evidence items in `validation.md` are satisfied
- [ ] All pre-release evidence items in `validation.md` are satisfied
- [ ] Human approver has reviewed change artifacts and issued approval token
- [ ] Approval token is recorded and linked in `traceability.md`

## AI Review Artifacts (CRITICAL)

- [ ] Governor review artifact exists, is timestamped, and records pass/fail for each checklist item
- [ ] Verifier review artifact exists, is timestamped, and records pass/fail for each checklist item
- [ ] Governor and Verifier review passes were executed as context-isolated invocations
- [ ] Review artifacts are stored in the change directory or designated review artifact location

## Library Extraction Acknowledgement (CRITICAL at governed tier)

- [ ] `/sds:extract-library-parts` extraction report has been reviewed
- [ ] Each candidate is either accepted for extraction or deferred with justification
- [ ] Deferred extractions are recorded in `library/extraction-backlog.yaml`

## Documentation Currency (ADVISORY)

- [ ] `proposal.md` still accurately describes the change as implemented
- [ ] `prd.md` requirements match what was actually built
- [ ] `architecture.md` reflects actual system topology after implementation
- [ ] Any deviations from original design are documented in `design.md`
